# Documentation Rules

## 코드 문서화
- 공개 API, 복잡한 로직에만 docstring/JSDoc을 작성한다.
- 자명한 코드에 불필요한 주석을 달지 않는다.
- TODO 주석에는 담당자와 이유를 명시한다: `# TODO(이름): 이유`

## README
- 프로젝트 설명, 설치 방법, 실행 방법을 포함한다.
- 예제 코드나 스크린샷이 있으면 더 좋다.

## 변경 기록
- 중요한 변경은 CHANGELOG.md에 기록한다.
- 형식: Keep a Changelog (https://keepachangelog.com)
