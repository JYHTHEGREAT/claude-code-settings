# Security Rules

## 금지 사항
- API 키, 비밀번호, 토큰을 코드에 하드코딩하지 않는다.
- 사용자 입력을 검증 없이 쿼리나 명령어에 직접 삽입하지 않는다.
- `eval()`, `exec()` 등 동적 코드 실행을 지양한다.

## 필수 사항
- 환경 변수 또는 시크릿 매니저를 통해 민감 정보를 관리한다.
- `.gitignore`에 `.env`, `credentials.json`, `*.pem` 등을 반드시 포함한다.
- 의존성 패키지의 알려진 취약점을 주기적으로 점검한다.

## 데이터 처리
- 개인정보가 포함된 데이터를 로그에 출력하지 않는다.
- 파일 경로 조작(path traversal) 공격을 방지한다.
